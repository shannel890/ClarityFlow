# Copilot Instructions for ClarityFlow

## Product Positioning
ClarityFlow should be presented as a healthcare workflow clarity concept focused on communication quality, documentation efficiency, and trust-centered UX.

## Implementation Priorities
- Keep architecture and naming product-oriented, not tutorial-oriented.
- Favor clear, maintainable Next.js + TypeScript patterns.
- Use Tailwind CSS for consistent, scalable UI styling.
- Keep Framer Motion usage minimal and purposeful.

## Communication Standards
- Write copy in a professional, healthcare-aware tone.
- Avoid overclaiming AI capabilities or clinical outcomes.
- Prioritize clarity, reliability, and operational realism.

## Portfolio Quality Bar
Every addition should reinforce:
1. Intentional product thinking
2. Healthcare workflow relevance
3. Trust-centered communication
4. Professional SaaS presentation
